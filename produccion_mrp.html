Agenda Maestra de Producción, MPS, MRP y JIT

Objetivo de aprendizaje:

Al finalizar este OVA, el estudiante será capaz de distinguir y aplicar los conceptos de agenda maestra de producción, planificación de necesidades de materiales (MPS), planificación de requerimientos de materiales (MRP) y justo a tiempo (JIT) para la gestión eficiente de la producción.

Definición de agenda maestra de producción:

La agenda maestra de producción es un programa que especifica la cantidad de cada producto final que se va a fabricar en cada período de tiempo (normalmente semanas) durante un horizonte de planificación. Sirve como guía para planificar los materiales y capacidades necesarios.

Objetivos de la agenda maestra:

  • Suavizar fluctuaciones estacionales de la demanda
  • Optimizar la utilización de recursos y capacidad de producción
  • Coordinar inventarios, compras y producción

Planificación de Necesidades de Materiales (MPS):

El MPS es una técnica de planificación de producción que convierte la agenda maestra de producción en una lista detallada de necesidades de materiales y producción específicos por periodos de tiempo.

Objetivos del MPS:

  • Coordinar las actividades de producción
  • Planificar la cantidad y el momento de la producción
  • Planificar la adquisición de materiales

Definición de MPS:

El MPS es un plan de producción detallado que establece la cantidad de cada producto que se producirá en cada período de tiempo.

Ejemplo numérico de un MPS:

Semana Producto A Producto B Producto C
1 100 75 300
2 175 90 350

Planificación de Requerimientos de Materiales (MRP):

El MRP es un sistema para calcular las necesidades de materiales y los tiempos en que se necesitan, basándose en el plan de producción (MPS), las listas de materiales (BOM) y los registros de inventario.

Objetivos del MRP:

  • Mantener los niveles de inventario lo más bajos posible
  • Planificar las actividades de fabricación, entrega y compra
  • Planificar la capacidad de producción necesaria para completar las actividades de fabricación a tiempo

Definición de MRP:

El MRP es un sistema de control de inventario y producción que utiliza listas de materiales, inventario y la agenda maestra para calcular las necesidades de materiales y los tiempos en que se necesitan.

Justo a Tiempo (JIT):

El JIT es una estrategia de gestión de inventario que se esfuerza por mejorar la eficiencia de una empresa y reducir los desechos al recibir bienes solo cuando se necesitan en el proceso de producción, reduciendo así los costos de inventario.

Objetivos del JIT:

  • Eliminar el desperdicio
  • Reducir el tiempo de espera
  • Reducir los niveles de inventario
  • Mejorar la eficiencia de los procesos

Definición de JIT:

El JIT es una estrategia de producción que se esfuerza por mejorar la eficiencia y la calidad de una empresa al producir y recibir bienes justo cuando se necesitan en el proceso de producción.

Introducción a la Planificación de Requerimientos de Materiales (MRP)

Una guía para entender y aplicar la MRP con Python y Pyomo

¿Qué es MRP?

La Planificación de Requerimientos de Materiales (MRP) es un sistema para gestionar y planificar las operaciones de producción y los inventarios en una empresa.

Conceptos Clave en MRP

  • Demandas: La cantidad de producto requerida.
  • Ofertas: La cantidad de producto disponible.
  • Restricciones: Las limitaciones en el proceso de producción.
  • Función objetivo: El objetivo que se busca alcanzar a través de la optimización, en este caso, minimizar el costo.

Métodos de Solución para MRP

  • Programación lineal: Un enfoque matemático para optimizar una función objetivo bajo ciertas restricciones.
  • Heurísticas: Estrategias basadas en reglas prácticas para encontrar soluciones aceptables en problemas complejos.
  • Algoritmos genéticos: Técnicas basadas en la selección natural para encontrar soluciones óptimas.

Aplicación con Python y Pyomo

Python es un lenguaje de programación potente y Pyomo permite modelar problemas de optimización de una manera sencilla.

import pandas as pd

Creamos una instancia del modelo concreto que servirá como base para definir variables, restricciones y la función objetivo.

Fragmento 4: Definición de Variables


model.x = Var((1,2,3), (1,2,3,4,5), within=NonNegativeIntegers)

Definimos las variables que representan las cantidades de productos a producir en cada periodo de tiempo.

Fragmento 5: Definición de la Función Objetivo


model.obj = Objective(expr = sum(c[i-1][j-1]*model.x[i,j] for i in range(1,4) for j in range(1,6)), sense=minimize)

Establecemos la función objetivo para minimizar el costo total de producción, utilizando los costos y las variables definidas anteriormente.

# (Añade aquí el código para definir las restricciones de demanda y oferta)

En este paso, definimos las restricciones que deben cumplir las variables, basándonos en las demandas y ofertas dadas.

Código Completo


		model = ConcreteModel()

		# Definiendo los índices para las variables
		model.i = RangeSet(0, 2)
		model.j = RangeSet(0, 4)
		
		# Definiendo las variables como enteras y no negativas
		model.x = Var(model.i, model.j, within=NonNegativeIntegers)
		
		# Definiendo la función objetivo para minimizar el costo total de producción
		model.obj = Objective(expr=sum(c[i][j] * model.x[i,j] for i in model.i for j in model.j), sense=minimize)
		
		# Definiendo las restricciones de demanda y oferta
		def demand_constraint(model, j):
			return sum(model.x[i, j] for i in model.i) == d[j]
		model.demand_constraint = Constraint(model.j, rule=demand_constraint)
		
		def offer_constraint(model, i):
			return sum(model.x[i, j] for j in model.j) == o[i]
		model.offer_constraint = Constraint(model.i, rule=offer_constraint)
